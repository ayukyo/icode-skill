# 步骤 6 — 终极终审 + 出具报告 + 统一修复

**命令**: `/icode audit`
**产出**: `{ICODE_OUT_DIR}/06_audit.md` + `{ICODE_OUT_DIR}/06_fixes.log`
**模型**: 全流程模式 `opus`；分步模式使用当前会话模型

## 前置校验

检查 `{ICODE_OUT_DIR}/03_plan_final.md` 和步骤4创建的代码文件是否存在，缺失则报错并提示先执行 `/icode code`。

## 6.1 出具终审报告

### ⚠️ 强制规则：禁止主 Agent 直接终审

终审报告**必须由子 Agent 独立完成**。主 Agent 只负责确定目录、读取文件、启动子 Agent、写文件、执行修复。**主 Agent 不得自行撰写终审报告或评估缺陷。**

1. 执行目录管理中的「检测最新目录」逻辑，确定 `ICODE_OUT_DIR`
2. 读取 `{ICODE_OUT_DIR}/03_plan_final.md` 和所有代码文件（从 `.ico_metadata.json` 的 `code_files` 获取列表）
   - 额外读取 `{ICODE_OUT_DIR}/05_review_rounds.json`（若存在），提取历史问题和角度汇总，避免重复发现已知问题
3. 按「通用规则」确定当前模型
4. 输出模型确认：`▶ 步骤6 使用模型：{当前模型名称}`
5. **启动子 Agent** 执行终审，prompt 为：

```
当前使用模型：{当前模型名称}。

请对照定稿计划，对已实施完成的代码做最终权威终审核验。

定稿计划：
{读取 {ICODE_OUT_DIR}/03_plan_final.md 的全部内容}

代码文件列表及内容：
{读取代码文件列表，逐个列出文件路径和内容}

**步骤5 深检历史（若存在）**：
{读取 `{ICODE_OUT_DIR}/05_review_rounds.json`，汇总已检查角度和已发现问题}

**注意**：已修复的深检问题无需重复报告。专注于深检未覆盖的角度或更深层次的架构问题。

**反偷懒机制**：代码简洁或编译通过不代表审计通过。审计核心是"代码是否严格按计划实施"。**必须先建立计划-代码追溯矩阵**（逐条对照计划中的功能点/接口/约束，找到代码中的对应实现位置），再逐维度评分。禁止跳过追溯直接给高分。

审核维度（6个，全部覆盖，缺一不可）：
1. **实施完整度** — 计划中所有功能点100%落地，每个功能点必须给出代码证据位置
2. **执行精准度** — 实现与计划一致，偏差处必须指出（文件+行号）
3. **方案偏离度** — 偏离项必须明确列出
4. **代码质量** — 可读性、性能、安全性
5. **跨文件一致性** — 接口变更全链路同步，上下游数据结构对齐
6. **残留风险** — 已知或潜在问题

执行流程：
1. 建立追溯矩阵 → 2. 基于矩阵逐维度评分 → 3. 对照计划校验项逐条勾对 → 4. 汇总问题清单 → 5. 给结论

文档格式：总体评分（百分制）+ 每维度独立评分评语 + 问题汇总清单（含位置、严重程度）+ 结论（通过/有条件通过/不通过）

直接输出终审报告，以 ===AUDIT START=== 开头、===AUDIT END=== 结尾。
```

6. **提取终审报告**（从 ===AUDIT START=== 到 ===AUDIT END===），写入 `{ICODE_OUT_DIR}/06_audit.md`

## 6.2 强制修复

1. 读取 `{ICODE_OUT_DIR}/06_audit.md` 中的问题清单
2. 按严重程度排序（高 → 中 → 低）逐个修复：
   - 每个问题用 Edit 工具修复
   - **修复时禁止删除现有注释**：仅修改问题相关的代码逻辑，保留原有注释内容
   - 修复后**必须使用 Write 工具追加记录到 `{ICODE_OUT_DIR}/06_fixes.log`**
3. 全部修复完毕后，做一次全局编译/语法验证（运行项目对应的编译命令，验证所有文件无错误），最多尝试 3 次
4. 确认所有问题已关闭
5. **更新 `{ICODE_OUT_DIR}/.ico_metadata.json`**：将 `status` 设为 `completed`，写回文件

## 6.3 最终交付

输出交付总结：

```

[✓] 步骤1: 计划拟定 — 完成
[✓] 步骤2: 计划审查 — 完成

[✓] 步骤4: 编码实施 — 完成
[✓] 步骤5: 循环复检 — 读取 .ico_metadata.json 中 deepcheck_total_rounds/deepcheck_clean_rounds
[✓] 步骤6: 终极终审 — 评分{分数}, {结论}
产出目录: {ICODE_OUT_DIR}/
```
