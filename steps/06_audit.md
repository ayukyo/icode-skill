# 步骤 6 — 终极终审 + 出具报告 + 统一修复

**命令**: `/icode audit`
**产出**: `{ICODE_OUT_DIR}/06_audit.md` + `{ICODE_OUT_DIR}/06_fixes.log`
**模型**: 全流程模式 `opus`；分步模式使用当前会话模型

## 前置校验

检查 `{ICODE_OUT_DIR}/03_plan_final.md` 和步骤4创建的代码文件是否存在，缺失则报错并提示先执行 `/icode code`。

## 6.1 出具终审报告

### ⚠️ 强制规则：禁止主 Agent 直接终审

终审报告**必须由子 Agent 独立完成**。主 Agent 只负责确定目录、读取文件、启动子 Agent、写入报告和修复。**主 Agent 不得跳过子 Agent 自行撰写终审报告或自行评估缺陷清单。**

1. 执行目录管理中的「检测最新目录」逻辑，确定 `ICODE_OUT_DIR`
2. 读取 `{ICODE_OUT_DIR}/03_plan_final.md`、`.ico_metadata.json` 获取代码文件列表和复检轮次数据（`deepcheck_total_rounds`、`deepcheck_clean_rounds`）
3. 按「通用规则」确定当前模型
4. **必须启动子 Agent 执行终审**（主 Agent 不可自行终审），按「通用规则」启动子 Agent，prompt 为：

```
当前使用模型：{当前模型名称}。
**请先进行深度思考（对照计划逐维度审核→独立评分→汇总问题→给出结论），再开始执行。**

请对照原始定稿计划，对当前已实施完成的代码做最终权威终审核验。

**输出方式（必须遵守）**：
1. **尝试**使用 Write 工具将内容写入 `{ICODE_OUT_DIR}/06_audit.md`（失败则忽略）
2. **必须在回复中输出完整终审报告**（主 Agent 会提取写入文件）
3. 回复以"===AUDIT START==="开头、以"===AUDIT END==="结尾

原始定稿计划：
{读取 {ICODE_OUT_DIR}/03_plan_final.md 的内容}

当前代码：
{读取 {ICODE_OUT_DIR}/.ico_metadata.json 中的 code_files 字段，列出所有代码文件路径和内容}

**硬性要求：必须逐项审核全部 6 个维度，每个维度必须独立评分和评语，不得跳过或遗漏。**

**反偷懒机制**：代码简洁或编译通过不代表审计通过。审计的核心不是"代码能不能跑"，而是"代码是否严格按计划实施"。**必须先建立追溯矩阵逐条对照计划，再逐维度评分，禁止跳过追溯直接给高分。**

审计维度（必须全部覆盖，缺一不可）：
1. **实施完整度** — 计划中所有功能点是否100%落地，**每个功能点必须给出对应的代码证据位置**
2. **执行精准度** — 实现是否与计划描述一致，**偏差处必须具体指出（文件+行号）**
3. **方案偏离度** — 是否偏离原定方案，偏离是否合理，**偏离项必须明确列出**
4. **代码质量** — 可读性、性能、安全性
5. **跨文件一致性** — 接口变更是否全链路同步、上下游数据结构是否对齐、修改的文件是否遗漏关联文件同步
6. **残留风险** — 还有哪些已知或潜在问题

执行流程（必须严格按序执行，不可跳过）：
1. **第一步：建立计划-代码追溯矩阵**，逐条列出定稿计划中的每个功能点/接口/约束，找到代码中的对应实现位置（文件名+行号），标记完成状态
2. **第二步：逐维度审核**，基于追溯矩阵对维度1-3评分（计划一致性），再对维度4-6评分（实现质量）
3. **第三步：对照计划中的校验项清单**，逐条勾对每个检查点是否通过
4. 汇总所有问题点清单
5. 给出最终结论

输出完整正式终审审查文档。

文档格式要求：
- 总体评分（百分制）
- 每个维度独立评分和评语
- 所有问题点汇总清单（含文件位置、严重程度）
- 最终结论（通过/有条件通过/不通过）
```

5. **提取子 Agent 回复中的终审报告内容**（从 ===AUDIT START=== 和 ===AUDIT END=== 之间提取），使用 Write 工具写入 `{ICODE_OUT_DIR}/06_audit.md`

## 6.2 强制修复

1. 读取 `{ICODE_OUT_DIR}/06_audit.md` 中的问题清单
2. 按严重程度排序（高 → 中 → 低）逐个修复：
   - 每个问题用 Edit 工具修复
   - **修复时禁止删除现有注释**：仅修改问题相关的代码逻辑，保留原有注释内容
   - 修复后**必须使用 Write 工具追加记录到 `{ICODE_OUT_DIR}/06_fixes.log`**
3. 全部修复完毕后，做一次全局编译/语法验证（运行项目对应的编译命令，验证所有文件无错误），最多尝试 3 次
4. 确认所有问题已关闭
5. **更新 `{ICODE_OUT_DIR}/.ico_metadata.json`**：将 `status` 设为 `completed`，写回文件

```
产出目录: {ICODE_OUT_DIR}/
```
